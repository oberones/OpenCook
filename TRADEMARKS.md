# OpenCook Trademark Policy

This policy describes acceptable use of the OpenCook name and related project
marks. It is intended to protect users from confusion while still allowing the
community to talk about, redistribute, package, and build integrations for the
project.

OpenCook is being prepared as a project trademark. The OpenCook name is used as
a mark by the OpenCook maintainers. Do not use the registered trademark symbol
unless and until a registration is issued.

The Apache License, Version 2.0, licenses copyright and patent rights in the
software. It does not grant trademark rights except for reasonable and customary
use in describing the origin of the software and reproducing the NOTICE file.

## Permitted Uses

You may use the OpenCook name without additional permission when the use is
truthful, non-confusing, and does not imply endorsement:

- Referring to the OpenCook project itself.
- Redistributing unmodified OpenCook source or binaries.
- Publishing patches, plugins, integrations, or documentation that clearly state
  they are for OpenCook or compatible with OpenCook.
- Using phrases like "for OpenCook", "works with OpenCook", or
  "OpenCook-compatible" when they accurately describe interoperability.
- Naming package metadata, dependency declarations, or build instructions that
  identify this project as the source.

## Uses That Need Permission

Please get written permission from the maintainers before using OpenCook in ways
that could make people think your work is the official project or is endorsed by
the project:

- Product, company, service, domain, social-account, event, or conference names.
- Modified distributions that use OpenCook as the primary product name.
- Logos, icons, or visual branding that are confusingly similar to official
  OpenCook branding.
- Merchandise, sponsorships, certification programs, or paid support branding.
- Statements that imply endorsement, certification, partnership, or official
  status.

## Forks And Modified Distributions

Forks and modified distributions should use their own distinct name. It is fine
to say that your work is "based on OpenCook" or "derived from OpenCook" as long
as that statement is accurate and not presented as official OpenCook branding.

If you distribute modified binaries, rename the product or make the modification
status prominent enough that users will not confuse it with an official
OpenCook release.

## Marking Guidance

Use `OpenCook` for code identifiers, commands, package paths, API examples,
filenames, and other technical contexts.

For public prose, use `OpenCook(tm)` or `OpenCook` on first prominent reference
where marking would be awkward in plain text. Do not use the registered
trademark symbol unless a registration has issued.

## Third-Party Marks

Chef, Chef Infra, Cinc, and any other third-party marks belong to their
respective owners. OpenCook is not affiliated with, endorsed by, or sponsored by
those owners unless the project explicitly says otherwise.

## Questions

If you are unsure whether a use is allowed, open an issue or contact the
maintainers through the project repository before publishing the use.
